The query uses `DeviceEvents` with `ActionType == "PlistPropertyModified"` — the correct table for macOS plist changes, as `DeviceFileEvents` does not surface these events. A `let` statement pre-filters to macOS devices via `DeviceInfo.OSPlatform` (the only table that exposes this field), then filters on plist paths containing `LaunchAgents` or `LaunchDaemons` to scope to persistence-relevant locations. The `AdditionalFields` JSON is parsed to surface the specific plist path, the property that was modified, and its new value, while `InitiatingProcess*` fields identify the responsible process, its command line, account, and hash for triage.
